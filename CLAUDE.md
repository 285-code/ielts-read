# IELTS-read

IELTS 精读学习工具：网页工作台 + Obsidian 联动

## 技术栈
- 纯前端 HTML/CSS/JS（单文件 `reader.html`）
- DeepSeek API（AI Chat，流式输出）
- Obsidian 兼容（导出 .md 丢进 `01-每日精读/`）
- localStorage 持久化（所有数据）

## 启动
```bash
# 直接双击 reader.html 即可
# 如需 AI Chat，建议用本地服务器打开（CORS 问题）：
python -m http.server 8080
```

## 交互规则

### 操作分层
| 操作 | 结果 |
|------|------|
| 单击一个单词 | 弹出统一浮窗：生词标注（红/黄/绿）+ 语法标注（8 种） |
| 浮窗内点「多词模式」→ 再点第二个词 | 选中词组范围，对该词组标注语法 |
| 点击段落 [I][II] | 整段显示在侧栏，可「整段问 AI」 |
| 点击句子 [n] 或 hover 句子 | 侧栏显示：句子原文 + 收藏按钮 + 问 AI + 「我的理解」笔记框 |
| Escape / 点击空白 | 关闭浮窗 |

### 视觉规则（注意：和早期设计相反）
- **生词**：**背景高亮**（淡粉/淡黄/淡绿），无下划线 —— CSS class `vocab-red/yellow/green`
- **语法标注**：**下划线**（8 种颜色，2px 粗），无背景 —— CSS class `syn-group syn-{type}`
- 两者互不干扰，一个词可以同时有生词背景 + 语法下划线

### 标记删除（均为 toggle）
- 生词：点已标记单词 → 弹出显示「已标 🔴 + ✕ 移除」，点 ✕ 删除
- 语法：同一位置再次点同个类型按钮 → 取消标记
- 词组语法：选中范围再次点同类型 → 取消该组标记

### 右侧面板（5 个 Tab）
- **Analysis**：句子/段落详情，「我的理解」笔记框（输入自动保存 2s debounce）
- **Vocab**：生词列表，按等级筛选（All/🔴/🟡/🟢），按红→黄→绿排序
- **Sentences**：收藏的句式列表，显示 Imitation / Paraphrase / 标签
- **Review**：间隔复习（SRS），到期词一目了然，弹窗模式测试
- **AI Chat**：DeepSeek API 流式对话，支持段落/句子一键发问，对话历史持久化

### SRS 复习系统
- 每个生词有 stage (0-5)，对应间隔 [0,1,3,7,14,30] 天
- 答对 stage+1，答错 stage-1（最低 0，间隔 1 天）
- 每天新词不超过当日标记数量，新词当天就会进入复习队列
- 复习队列随机打乱，含到期词 + 今日新词

### 句式收藏
- 收藏句子时可选标签：功能标签（开头句/结尾句/主题句...）+ 结构标签（简单句/复合句/倒装句...）+ 话题标签（科学/历史/教育...）
- 必填 Imitation（仿写）或 Paraphrase（改写）之一
- 导出 .md 时一并输出

### 文章管理
- 导入：标题 + 来源代号 + 粘贴原文（自动按空行分段，每段自动拆句）
- 编辑：点「编辑文章」可修改标题/来源/正文，附带「自动校对」按钮
- 自动校对（10 步）：0→O 混淆 / 1→l 混淆 / rn→m / 词内多余空格 / 全角标点→半角 / 标点前多余空格 / 句号后缺空格 / 双空格 / camelCase 拆分 / 词典分割粘连词（≥13 字符）

### 导出
- **Export .md**：日期命名，包含句子理解笔记 + 生词表（含原句） + 句式收集（含仿写/改写/标签）
- **Export JSON**：全量数据备份（文章/生词/句式/语法标注/笔记）
- **Import JSON**：合并导入（去重，不覆盖已有数据）
- 文件直接丢进 `01-每日精读/` 即可在 Obsidian 打开

### 数据持久化
- localStorage key 列表：
  - `ielts_article` — 当前文章
  - `ielts_vocab` — 生词数组
  - `ielts_sentences` — 收藏句式数组
  - `ielts_syntax` — 语法标注数组
  - `ielts_sentence_notes` — 句子笔记对象 {sentId: text}
  - `ielts_api_key` — DeepSeek API Key
  - `ielts_chat_history` — AI 对话历史
- 清空数据用右下角「Clear」按钮
- Claude 每次新对话不会记住历史，需看本文件恢复上下文
